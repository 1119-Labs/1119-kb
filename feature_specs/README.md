# Feature specs

Each subfolder in `feature_specs/` is one feature. It must contain a detailed implementation doc (e.g. `implementation.md` or `README.md`) with:

- **Steps to implement** — Numbered steps, file paths, and key code touchpoints.
- **How to test** — Automated tests if possible; otherwise clear manual testing steps and expected outcome.

Create the folder and doc in plan mode before implementing the feature. See `.cursor/rules/features.mdc` for the full rule.
